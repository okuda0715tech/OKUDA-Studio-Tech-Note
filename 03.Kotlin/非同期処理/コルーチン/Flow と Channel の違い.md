# Flow と Channel の違い

## 目的

- Flow
  - データストリームからのデータの取得
- Channel
  - 異なるコルーチン間でのデータの受け渡し


## 宣言的か命令的か

- Flow
  - 宣言的
- Channel
  - 命令的


### 命令的な例

```kotlin
val list = mutableListOf<Int>()
for (i in 1..5) {
    list.add(i * 2)
}
```


### 宣言的な例

宣言的な記述は、変数に代入しやすかったり、再利用しやすいことが特徴らしい...。が、宣言的と命令的の明確な違いの説明は、なかなか難しい...。

```kotlin
val list = (1..5).map { it * 2 }
```
